External API Developer Guide
This guide covers the public 69labs external API for text-to-speech, SRT subtitle generation, image generation, video generation, motion graphics, and utility tools. It documents the current request and response contract, including TTS censorship recovery, model discovery, polling, downloads, image compression, and common error cases.

Base URL: https://69labs.vip/api/v1

Quick Start
Generate your first image in three steps:

1. Get your API key
Go to Settings > API Keys in the 69labs web UI and create a key. It starts with vk_.

2. Generate an image
curl -X POST https://69labs.vip/api/v1/images/generate \
  -H "Authorization: Bearer vk_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "A futuristic city skyline at sunset"}'
Response (HTTP 201):

{
  "id": "cm7abc123def",
  "queuePosition": 2
}
3. Poll until complete, then download
# Poll status
curl https://69labs.vip/api/v1/images/status/cm7abc123def \
  -H "Authorization: Bearer vk_your_api_key"

# Once status is "COMPLETED", download
curl -L -o image.png https://69labs.vip/api/v1/images/download/cm7abc123def \
  -H "Authorization: Bearer vk_your_api_key"
That's it. The rest of this guide covers every option in detail.

Integration Architecture Patterns
Use the public REST API directly from trusted server-side code, not from a public browser bundle with your shared API key.

Scenario	Recommended pattern
Public website or SaaS app	Browser calls your backend; your backend calls /api/v1 with the vk_* key
Full backend service	Store local jobs, submit to 69labs from a worker, poll with backoff, then copy outputs to your own storage
Serverless or automation	Store the returned job ID durably, resume polling from a scheduled function, and respect Retry-After
Batch workflow	Use bulk endpoints with idempotencyKey, then poll batch items and dispatched jobs
LLM-generated client	Give the LLM /api/docs/openapi.yaml and require REST-only code, runtime model discovery, async polling, and explicit error handling
Thin backend proxy
app.post("/api/images", async (req, res) => {
  const response = await fetch("https://69labs.vip/api/v1/images/generate", {
    method: "POST",
    headers: {
      Authorization: "Bearer " + process.env.LABS_API_KEY,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      prompt: String(req.body.prompt || "").slice(0, 15000),
      model: req.body.model,
      aspectRatio: req.body.aspectRatio,
    }),
  });

  const data = await response.json();
  res.status(response.status).json(data);
});
Store the returned job ID with your own user or tenant record. Let your frontend poll your backend, and let your backend poll 69labs. This keeps API keys private and gives you one place to enforce user permissions, rate limits, and logging rules.

Full backend worker
For production queues, store a local job row with your user ID, requested capability, sanitized request metadata, the returned 69labs job ID, local status, last error code, and next poll time. A worker should submit the REST request, save the 69labs job ID immediately, poll status with backoff, follow download redirects on completion, and copy outputs into your own storage when you need long-term access.

LLM integration rule
When asking an LLM to write integration code, tell it: use only https://69labs.vip/api/v1, read https://69labs.vip/api/docs/openapi.yaml, discover models/templates at runtime, treat generation as asynchronous, respect Retry-After, follow download redirects, and never call JWT, settings, admin, worker, or internal web routes.

Authentication
All endpoints require an API key in the Authorization header:

Authorization: Bearer vk_your_api_key
Keys always start with the vk_ prefix
Each user can have one active key at a time
Keys are capability-scoped — your account must have the required TTS, image, and/or video API access enabled
Auth errors:

HTTP Status	Code	Cause
401	UNAUTHORIZED	Missing, malformed, invalid, inactive, or deactivated key; the public response is always generic
403	FORBIDDEN	Key is valid but access to the requested capability is denied

Each model object tells you exactly what parameters it accepts:

{
  "id": "flux-2-pro",

  "resolutionCosts": {
    "1k": 1,
    "2k": 2

  "aspectRatios": [
    { "value": "1:1", "label": "1:1 (Square)" },
    { "value": "16:9", "label": "16:9 (Landscape)" },
    { "value": "9:16", "label": "9:16 (Portrait)" }
  ],
  "resolutions": [
    { "value": "1k", "label": "1K" },
    { "value": "2k", "label": "2K" }
  ],
  "qualityOptions": null,
  "supportsImageInput": true,
  "maxImageUrls": 8,
  "defaults": {
    "aspectRatio": "1:1",
    "resolution": "1k"

Key fields:

Field	Meaning
cost	Flat credits consumed per generation
resolutionCosts	Optional per-resolution credit overrides. When present, these override the flat cost
aspectRatios	Valid aspectRatio values for this model
resolutions	Valid resolution values (null if not supported)
durations	Valid duration values — video models only (null if not supported)
modes	Valid mode values — video models only (null if not supported)
supportsImageInput	Whether the model accepts imageUrls
maxImageUrls	Max number of reference images (null if no image input)
defaults	Default values used when you omit optional parameters
For pay-per-use accounts, model responses can also include:

billingMode: "PAY_PER_USE"
retention.generationRetentionHours with the account's output download window
credits: null
enabled, pricing, limits, and richer monthlyUsage on image/video/motion sections
ppuPriceCents on each available image/video model after account-level allow/deny rules are applied
tts setup on GET /models with billing mode, rate, concurrency, current-month characters/jobs, cap, and public voice/model guidance
srtSubtitles setup on GET /models with enabled state, billing mode, per-minute/per-job prices, and current-month minute/job usage
There is no public TTS model catalog endpoint. For TTS, submit voiceId and optionally modelId directly to /tts/generate. Catalog voices can be listed with GET /tts/minimax/voices.

Pay-Per-Use Usage Lookup
Pay-per-use API keys can read summary ledger totals with GET /usage.

curl "https://69labs.vip/api/v1/usage?startDate=2026-05-01&endDate=2026-05-31" \
  -H "Authorization: Bearer vk_your_api_key"
Omit both dates to use the current UTC month through today. Custom ranges must provide both startDate and endDate, use inclusive UTC YYYY-MM-DD values, and span no more than 366 days.

Tools
Tools are synchronous utility endpoints. The image compressor accepts one uploaded image and returns image bytes immediately; it does not create an async job.

curl -X POST https://69labs.vip/api/v1/tools/image-compressor/compress \
  -H "Authorization: Bearer vk_your_api_key" \
  -F "mode=lossy" \
  -F "file=@thumbnail.jpg" \
  -o thumbnail-compressed.jpg
Supported uploads are JPEG, PNG, WebP, and AVIF up to 10 MB. mode is lossy or lossless; lossy is the default and targets high-quality smaller output. The endpoint spends the configured TOOL_CREDITS action cost for regular credit accounts. Pay-per-use tool billing is not enabled yet.

Useful response headers:

Header	Meaning
X-Tool-Cost	Tool credits charged
X-Original-Size-Bytes	Uploaded size
X-Compressed-Size-Bytes	Returned size
X-Saved-Bytes	Bytes saved
X-Saved-Percent	Rounded percent saved
X-Output-Filename	Suggested output filename
X-Used-Original	true if the original was returned because compression was not smaller
Text-to-Speech
Lifecycle
POST /tts/generate  →  GET /tts/status/{id}  →  GET /tts/download/{id}
     (submit)               (poll)                    (download)
Recoverable censored flow:

PROCESSING  →  CENSORED  →  POST /tts/retry-censored/{id}  →  PROCESSING
Request parameters
Parameter	Type	Required	Description
text	string	Yes	1–500000 characters
voiceId	string	Yes	Voice ID for the selected voiceProvider; use the full catalog voice_id for catalog voices
voiceName	string	No	Optional display name
voiceProvider	string	No	elevenlabs, edgetts, or minimax; these are wire-level API values. edgetts uses discounted TTS credits; defaults to elevenlabs
modelId	string	No	Defaults to eleven_multilingual_v2; catalog voice mode defaults to speech-02-hd and accepts speech-2.8-hd / speech-2.8-turbo; edgetts uses edge-tts
splitOutput	boolean	No	Return split outputs instead of a single merged file
voiceSettings	object	No	stability, similarityBoost, speed, style, useSpeakerBoost
minimaxSettings	object	No	Catalog voice mode only: speed 0.01–10, pitch -12 to 12, volume 0.5–2, languageBoost (defaults to auto; ISO hints normalize server-side)

curl -X POST https://69labs.vip/api/v1/tts/generate \

SRT subtitles
Completed TTS jobs can be transcribed into SRT subtitle files through the public API. SRT generation is asynchronous and starts after the audio job is COMPLETED.

POST /tts/generate -> GET /tts/status/{id} -> POST /tts/generate-srt/{id}
  -> GET /tts/srt-status/{id} -> GET /tts/download-srt/{id}
Request subtitles:

curl -X POST https://69labs.vip/api/v1/tts/generate-srt/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer vk_your_api_key"
{
  "jobId": "123e4567-e89b-12d3-a456-426614174000",
  "srtStatus": "PENDING",
  "creditCost": 150
}
Pay-per-use SRT responses use creditCost: null and include billing metadata:

{
  "jobId": "123e4567-e89b-12d3-a456-426614174000",
  "srtStatus": "PENDING",
  "creditCost": null,
  "billingMode": "PAY_PER_USE",
  "ppuBillableUnits": 1,
  "ppuBillingUnit": "SRT_JOB",
  "ppuSrtMinutes": 5,
  "ppuUnitPriceCents": 150,
  "ppuAmountCents": 150
}
Poll SRT status:

curl https://69labs.vip/api/v1/tts/srt-status/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer vk_your_api_key"
{
  "id": "123e4567-e89b-12d3-a456-426614174000",
  "srtStatus": "COMPLETED",
  "srtError": null,
  "srtFileAvailable": true,
  "creditCost": 0
}
Download the subtitle file:

curl -L -o subtitles.srt https://69labs.vip/api/v1/tts/download-srt/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer vk_your_api_key"
SRT generation is free for credit-based accounts when the active tier has subtitles enabled. Finite tiers count accepted subtitle jobs against the current UTC month's allowance; failed subtitle jobs do not count. Pay-per-use SRT still requires SRT enabled on the account and can bill per started minute or per subtitle job while tracking subtitle duration. If generation fails, PPU reservations are released. Audio and SRT files share the same output retention window; expired files return 410.

Fixing censored TTS jobs
When TTS status is CENSORED, the response can include:

blockedChunks: array of { index, text }
retryExpiresAt: 6-hour retry deadline for chunked jobs
chunksTotal and chunksCompleted for chunked progress
Retry example:

curl -X POST https://69labs.vip/api/v1/tts/retry-censored/cm7tts123abc \
  -H "Authorization: Bearer vk_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "correctedChunks": [
      {
        "index": 2,
        "text": "Rewritten text that avoids the content moderation block"
      }
    ]
  }'
Rules:

Corrected text is capped at 5000 characters per chunk
Chunked censored jobs have a 6-hour retry window
A retried job can become CENSORED again if the rewritten text is still blocked
Voice Clone TTS
Use cloned voices for TTS generation. Voice creation is UI-only — these endpoints let you list existing clones and generate audio with them.

Lifecycle
GET /voice-clones            →  browse your voices
GET /voice-clones/library    →  browse global voices
POST /voice-clones/generate  →  GET /tts/status/{id}  →  GET /tts/download/{id}
Voice clone TTS jobs are standard TTS jobs — poll and download using the regular TTS endpoints.

List your cloned voices
curl https://69labs.vip/api/v1/voice-clones \
  -H "Authorization: Bearer vk_your_api_key"
Response:

{
  "voiceClones": [

      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "name": "My Voice",
      "language": "English",
      "gender": "male",
      "previewUrl": "https://...",
      "isGlobal": false,
      "status": "ready",
      "createdAt": "2026-04-09T12:00:00.000Z"
    }
  ]
}
List global voice library
curl https://69labs.vip/api/v1/voice-clones/library \
  -H "Authorization: Bearer vk_your_api_key"
Get a single voice clone
curl https://69labs.vip/api/v1/voice-clones/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer vk_your_api_key"
Returns voices you own or global library voices.

Generate TTS with a cloned voice
curl -X POST https://69labs.vip/api/v1/voice-clones/generate \
  -H "Authorization: Bearer vk_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "voiceCloneId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "text": "Hello from my cloned voice.",
    "model": "speech-02-hd",
    "speed": 1.0
  }'
Request parameters
Parameter	Type	Required	Description
voiceCloneId	string (UUID)	Yes	Voice clone ID (owned or global)
text	string	Yes	1–100000 characters
model	string	No	Defaults to speech-02-hd; accepts speech-2.8-hd and speech-2.8-turbo
speed	number	No	0.5–2 speed multiplier
pitch	number	No	-12 to 12 semitones
volume	number	No	0–10
language_boost	string	No	Language hint such as "English" or "Turkish"; defaults to auto; ISO hints normalize server-side
Access rules
Your clones: GET /voice-clones lists voices you created
Global voices: GET /voice-clones/library lists admin-curated voices available to everyone
Generate: Accepts both your own clone IDs and global library IDs
Requires voice cloning: Your tier must have voice cloning enabled. Listing works regardless, but generate returns 403 without it.
Credits
Voice clone TTS consumes advance voice credits (cost = text character count), same as the web UI. Credits are refunded on job failure.

Image-to-image (reference images)
Models with supportsImageInput: true accept reference image URLs. The images must be publicly accessible. Use imageUrls for new integrations; legacy clients may send one unambiguous alias (referenceImage, imageUrl, referenceImageUrls, or referenceImages), which is normalized to imageUrls. Do not combine reference fields in one request.

curl -X POST https://69labs.vip/api/v1/images/generate \

Parameter	Type	Required	Description
prompt	string	Yes	1–15000 characters

aspectRatio	string	No	Must be valid for the model
resolution	string	No	Only for models with resolutions

Polling for Completion
After submitting a job, poll the status endpoint until it reaches a terminal state.

Status transitions
PENDING  →  PROCESSING  →  FINALIZING  →  COMPLETED
                      ↘              ↘
                        CENSORED       FAILED
         →  CANCELLED
Status	Meaning
PENDING	Queued, waiting to be picked up
PROCESSING	Actively generating output
FINALIZING	Generation done, file being stored
COMPLETED	Ready to download
FAILED	Generation failed (credits are refunded)
CANCELLED	Job was cancelled
CENSORED	TTS only. One or more chunks were blocked and must be rewritten before retry
Polling example
# Poll every 3 seconds
while true; do
  STATUS=$(curl -s https://69labs.vip/api/v1/images/status/cm7abc123def \
    -H "Authorization: Bearer vk_your_api_key" | jq -r '.status')

  echo "Status: $STATUS"

  if [ "$STATUS" = "COMPLETED" ] || [ "$STATUS" = "FAILED" ] || [ "$STATUS" = "CANCELLED" ]; then
    break
  fi

  sleep 3
done
async function pollUntilDone(jobId, type = "images") {
  const baseUrl = "https://69labs.vip/api/v1";
  const headers = { "Authorization": "Bearer vk_your_api_key" };

  while (true) {
    const res = await fetch(`${baseUrl}/${type}/status/${jobId}`, { headers });
    const { status } = await res.json();

    console.log(`Status: ${status}`);

    if (["COMPLETED", "FAILED", "CANCELLED", "CENSORED"].includes(status)) {
      return status;
    }

    await new Promise(resolve => setTimeout(resolve, 3000));
  }
}

Credits & Rate Limits
Credit system
Each generation consumes credits. There are two credit pools:

Hourly credits — Resets every hour. Shown in credits.resetsAt.
Monthly credits — Pro-rated grant based on your subscription. Shown in monthlyUsage.
The cost field on each model is the default per-generation cost. When a model also returns resolutionCosts, the selected resolution overrides the flat cost.

400	BAD_REQUEST	Invalid prompt, bad model ID, invalid aspect ratio, invalid censored retry payload, etc.	Check parameters against the endpoint contract and model capabilities
400	VALIDATION_ERROR	Authenticated request body or query param failed validation	Use details.fieldErrors to fix the fields

Retry strategy
async function fetchWithRetry(url, options, maxRetries = 3) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    const res = await fetch(url, options);

    if (res.ok) return res;

    if (res.status === 429) {
      const retryAfter = parseInt(res.headers.get("Retry-After") || "5", 10);
      console.log(`Rate limited. Retrying in ${retryAfter}s...`);
      await new Promise(r => setTimeout(r, retryAfter * 1000));
      continue;
    }

    if (res.status >= 500 && attempt < maxRetries) {
      const delay = Math.pow(2, attempt) * 1000; // 1s, 2s, 4s
      console.log(`Server error ${res.status}. Retrying in ${delay}ms...`);
      await new Promise(r => setTimeout(r, delay));
      continue;
    }

    const body = await res.json();
    throw new Error(`API error ${res.status}: ${body.code} - ${body.error}`);
  }
}

Tips & Best Practices
Check model capabilities before generating. Fetch /v1/images/models or /v1/videos/models and validate your parameters against the model's accepted aspectRatios, resolutions, durations, and modes.

Check credits before generating. If credits.remaining is 0, your job will be rejected with a 403. Use the resetsAt timestamp to schedule retries.

Handle failed jobs. Credits are refunded on failure. Don't retry immediately — if the prompt caused the failure, the retry will fail too. Adjust the prompt or try a different model.

Use resolutionCosts when present. Some image models have resolution-based pricing, so do not assume the flat cost applies to every resolution.

Follow redirects on download. The download endpoint typically returns HTTP 302. Use curl -L or set redirect: "follow" in fetch.

Download files within the retention window. PPU model discovery returns retention.generationRetentionHours; tier accounts use their configured tier retention. After expiration, the download endpoint returns 410 GONE.

Use exponential backoff for retries. On 429 (rate limit), respect the Retry-After header. On 5xx errors, use exponential backoff (1s, 2s, 4s).

Handle TTS CENSORED explicitly. Treat it as a recoverable state, inspect blockedChunks, rewrite only the flagged chunks, and resubmit them to /tts/retry-censored/{jobId}.

Don't poll too aggressively. Status endpoints allow 120 requests/min, but polling every 2–5 seconds for TTS, 3–5 seconds for images, and 5–10 seconds for videos is sufficient and avoids unnecessary load.
